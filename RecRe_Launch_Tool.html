Geographic Scheduling Rules
These values control what launch dates are offered on a new card and whether back-to-back launches can be linked. When a launch card is created, the tool calculates the customer's distance from HQ, assigns a tier, and surfaces only dates that meet that tier's lead time on the correct deploy day. All fields are structured inputs — no free text — so the rule engine can read them directly.
Tier 1 — Local
Anything under this = Tier 1
Earliest date offered from today
Dates offered on this day only
How many dates shown to customer
Tier 2 — Regional
Between Tier 1 max and this = Tier 2
Tier 3 — National
Anything beyond Tier 2 max = Tier 3
Back-to-Back Linking
If two launches fall within this distance, the tool flags them as linkable
Attendance Rules
When a launch card is created, the tool reads box count and building count from the deal and auto-sets the Attendance field. DLL can override on the card. "Technician" is role-agnostic — DLL decides Field Nation vs RecRe staff. Each rule below is a structured value the engine reads directly.
